## Тест план

## Цель

Проверить, что основные пользовательские сценарии работают корректно, найти дефекты, влияющие на удобство использования продукта, корректность и завершение заказа.

## Объект тестирования

- страница логина
- страница списка товаров (каталог)
- сортировка товаров
- страница карточки товара
- страница корзины
- страница оформления заказа
- страница подтверждения заказа
- страница подтверждения оплаты

## Виды тестирования

- функциональное тестирование
- негативное тестирование
- exploratory testing
- sanity-check UI/UX
- проверка граничных значений для полей ввода

## Тестовое окружение

- dekstop browser
- стандартное разрешение экрана
- стабильное инетернет-соединение

## Тестовые данные (login / password)

- `standard_user / secret_sauce`
- `locked_out_userr / secret_sauce`
- `problem_user / secret_sauce`
- `performance_glitch_user / secret_sauce`
- `error_user / secret_sauce`
- `visual_user / secret_sauce`

## Критерии входа

- приложение доступно
- тестовые данные подготовлены
- нет сбоев в окружениях

## Критерии выхода

- протестированы критические сценарии
- задокументированы блокирующие дефекты
- подготовленно тест-резюме


## Риски

- нестабильное окружение
- сломанные селекторы или изменчивый UI
- противоречивые тестолвые данные

## Артефакты

- чек-листы
- тест-кейсы
- баг-репорты
- тест-резюме
