## Тест план

## Цель

Проверить, что основные пользовательские сценарии работают корректно, найти дефекты, влияющие на удобство использования продукта, корректность и завершение заказа.

## Объект тестирования

- страница логина
- страница списка товаров (каталог)
- сортировка товаров
- страница карточки товара
- страница корзины
- страница оформления заказа
- страница подтверждения оплаты
- страница успешно оформленного заказа

### In-scope (что в рамках тестирования)

- авторизация
- каталог товаров
- сортировка товаров
- операции с корзиной 
- оформление заказа
- сообщения валидации
- базовая навигация

### Out-of-scope (что не в рамках тестирования)

- нагрузочное тестирование backend
- доступность внешних сервисов
- интеграция с платежным шлюзом
- security penetration testing (пентест)
- тестирование интеграции с платежным шлюзом

## Виды тестирования

- функциональное тестирование
- негативное тестирование
- exploratory testing
- sanity-check UI/UX
- проверка граничных значений для полей ввода

## Подход к тестированию

Проект тестируется с точки зрения пользователя с использованием функциональных, негативных и исследовательских сценариев (exploratory testing).

## Тестовое окружение

- desktop browser
- стандартное разрешение экрана
- стабильное интернет-соединение

## Критерии входа

- приложение доступно
- тестовые данные подготовлены
- нет сбоев в окружениях

## Критерии выхода

- протестированы критические сценарии
- задокументированы блокирующие дефекты
- подготовлено тест-резюме


## Риски

- нестабильное окружение
- сломанные селекторы или изменчивый UI
- противоречивые тестовые данные
